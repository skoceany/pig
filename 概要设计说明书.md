# 智能养殖猪病诊断智能体 - 概要设计说明书

## 1. 引言

### 1.1 编写目的
本文档基于需求规格说明书，对智能养殖猪病诊断智能体进行概要设计，包括系统架构、模块划分、接口设计等内容，为详细设计提供依据。

### 1.2 设计原则
- **模块化设计**：系统划分为多个独立模块，降低耦合度
- **可扩展性**：预留扩展接口，支持后续功能扩展
- **易用性**：提供简洁友好的用户界面
- **性能优先**：优化算法和数据处理流程，确保快速响应

## 3. 数据流设计

### 3.1 诊断流程数据流

```
用户上传图片 → 图片预处理 → 调用多模态模型 → 模型推理 → 结果解析 → 返回诊断结果
     ↑                                                        │
     └────────────────── 诊断历史存储 ←───────────────────────┘
```

### 3.2 数据处理流程

```
原始数据（图片+描述） → 数据清洗 → 数据标注 → 数据增强 → 模型训练 → 模型评估 → 模型部署
```

## 4. 接口设计

### 4.1 前端接口

| 接口名称 | 功能描述 | 参数 | 返回值 |
|----------|----------|------|--------|
| uploadImage | 上传诊断图片 | image: File | { success: boolean, message: string } |
| getDiagnosis | 获取诊断结果 | imageId: string | DiagnosisResult |
| getDiseaseInfo | 获取疾病详细信息 | diseaseName: string | DiseaseInfo |
| getHistory | 获取诊断历史 | page: number, size: number | PagedResult |
| deleteHistory | 删除诊断记录 | recordId: string | { success: boolean } |

### 4.2 后端API接口

| API路径 | HTTP方法 | 功能描述 |
|---------|----------|----------|
| /api/diagnosis | POST | 上传图片并进行诊断 |
| /api/disease/{name} | GET | 查询疾病详细信息 |
| /api/history | GET | 获取诊断历史列表 |
| /api/history/{id} | DELETE | 删除诊断记录 |

### 4.3 模型接口

| 接口名称 | 功能描述 | 输入 | 输出 |
|----------|----------|------|------|
| predict | 疾病预测 | image: numpy array | { disease: string, confidence: float } |
| generate_interpretation | 生成疾病解读 | disease: string, image_features: array | interpretation: string |

## 6. AI模型设计

### 6.1 模型架构

```
输入层：图片（RGB）+ 提示词
图像编码器：CLIP ViT-L/14
文本编码器：LLaMA-2 7B
交叉注意力层：融合图像和文本特征
解码器：生成诊断结果和疾病解读
```

### 6.2 模型训练流程

```
1. 数据准备：整理352张图片和对应的描述文本
2. 数据预处理：统一图片尺寸、数据增强、文本分词
3. 预训练：使用LLaVA预训练模型作为基础
4. 微调：使用猪病数据集进行微调
5. 评估：在测试集上评估模型性能
6. 优化：根据评估结果调整模型参数
```

### 6.3 提示词设计

```
诊断提示词模板：
"分析这张猪的照片，判断猪是否患病。如果患病，请识别具体疾病名称，并详细描述症状、病因和应对建议。"

疾病解读提示词模板：
"请详细解读{疾病名称}，包括：1.疾病概述 2.主要症状 3.传播途径 4.治疗方法 5.预防措施"
```

## 8. 安全性设计

### 8.1 数据安全
- 图片上传时进行病毒扫描
- 用户数据加密存储
- 定期备份数据库

### 8.2 接口安全
- API接口添加访问频率限制
- 关键接口添加身份认证
- 传输层使用HTTPS加密

### 8.3 模型安全
- 模型文件权限控制
- 模型推理服务隔离
- 防止模型被恶意攻击
